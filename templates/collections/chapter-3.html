Map Interface

Lưu trữ các phần tử theo cặp khóa - giá trị (key - value)

Map không chứa các khóa trùng lặp

Mỗi khóa (key) chỉ tham chiếu tương ứng 1 giá trị (value)

Có 3 lớp hiện thực

HashMap

LinkedHashMap

TreeMap

Trường hợp sử dụng:

Lưu trữ tập dữ liệu lớn và Không quan tâm thứ tự lưu trữ, HashMap để cho kết quả được nhanh nhất

Muốn đạt hiệu suất và giữ lại thứ tự phần tử khi thêm vào dùng LinkedHashMap

Trường hợp muốn các khóa key được tự động sắp xếp tăng dần dùng TreeMap

HashMap

public static void main(String[] args) {
    Map map = new HashMap();
    map.put("java", "Java Collections Framework");
    map.put("javascript", "JavaScript Pro");
    map.put("html", "html css pro");

    // không được trùng key, 
    // giá trị của key khai báo sau sẽ ghi đè giá trị đầu
    map.put("java", "Java SE");

    Iterator it = map.keySet().iterator();
    while(it.hasNext()) {
        Object key = it.next();
        Object value = map.get(key);
        System.out.println(key + " - " + value);
    }
}

delete method 

// key là java xóa tất
map.remove("java");

// Java SE vẫn còn tồn tại
map.remove("java", "Java Collections Framework");

Thứ tự thêm vào khác thứ tự in ra.

HashMap đã thay đổi thứ tự các phần tử. Nhằm mục đích truy cập các phần tử nhanh nhất có thể

Như vậy trong trường hợp chúng ta có một lượng lớn các phần tử cần lưu trữ mà chúng ta ưu tiên về tốc độ của hiệu suất truy cập thì ta nên sử dụng HashMap

LinkedHashMap

Thứ tự thêm vào như thế nào khi in ra sẽ giữ nguyên giống như vậy.

Không cho hiệu suất tốt bằng lớp hiện thực HashMap()

TreeMap

Tự động sắp xếp các giá trị key theo thứ tự tăng dần
