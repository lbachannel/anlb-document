Kiểu dữ liệu & biến

Biến là gì?

Biến là 1 vùng ram được đặt tên, chiếm 1 số byte nhất định tùy loại.

Dữ liệu, thông tin mà nó sẽ chứa bên trong

Vùng ram dùng để lưu trữ dữ liệu thông tin và xử lý.

Primitive Data Type

Có 8 loại dữ liệu nguyên thủy

Data type Số byte Số bit
byte 1 byte 8
short 2 byte 16
int 4 byte 32
long 8 byte 64
float 4 byte 32
double 8 byte 64
boolean không chỉ định rõ kích thước, hoặc 1 bit false/true
char 2 byte 16

Integer

int age = 20;
long m = 3000000000; // error
long m = 3000000000L; // correct

Mọi con số, kí tự xuất hiện trong code được gọi là literal value

Số nguyên xuất hiện trong code mặc định là int.

Double

Java ưu tiên double hơn float

float vat = 0.1; // error
float vat = 0.1F; // correct

Characters

char 2 byte, lưu bảng mã Unicode

char m = '$';

Boolean

boolean marriedStatus = false;
if (marriedStatus == true) {
    System.out.println("Đánh đồn có địch mới vui");
} else {
    System.out.println("Vườn hồng có lối nhưng chưa ai vào");
}

Biến var

JDK10 - LOCAL VARIABLE TYPE INFERENCE

Từ JDK10, Java hỗ trợ khả năng khai báo biến cục bộ trong hàm mà không cần chỉ rõ data type một cách tường minh - type inference (suy luận kiểu) bằng cách sử dụng từ khoá var.

Java Compiler sẽ dựa trên cách khai báo biến và gán giá trị để suy luận ra kiểu dữ liệu của biến.

Một số lưu ý khi dùng var

Bắt buộc phải gán giá trị cho biến khi khai báo có dùng var 

var x; // bị báo lỗi
var y = "Do IT Now"; // hợp lệ

Không thể gán giá trị null cho biến lúc khai báo 

var x = null; //bị báo lỗi

Không đổi kiểu dữ liệu ban đầu của biến sang kiểu khác 

var x = 10;
x = "Hello"; // //bị báo lỗi tương thích kiểu dữ liệu do x đã được xác định là int

JVM - JRE - JDK

Khi chạy chương trình Java sẽ thông qua 2 bước:

Các file code sẽ được compiler (biên dịch) thành các mã byte code dưới dạng file .class

Tiếp theo thì các mã byte code sẽ được thông dịch Interpreter thành mã máy machine code để máy có thể hiểu và thực thi chương trình.

Source code Compiler (biên dịch) Byte Code (.class) Interpreter (thông dịch) Machine Code (mã máy) Output

JVM

JVM: Java Virtual Machine (máy ảo Java)

JVM chịu trách nhiệm chuyển đổi các mã byte code thành mã máy tại thời điểm thực thi chương trình

Nhưng chỉ với JVM thôi thì chưa đủ. Thường chương trình Java chúng ta sẽ có thêm một số thư viện Libraries để chạy chương trình

Ví dụ: Java Standard Library:

+java.lang

+java.util

+java.io ....

JRE

JRE: Java Runtime Enviroment (Môi trường thực thi Java)

JRE = JVM + Libraries

Về cơ bản nếu chỉ muốn chạy chương trình Java thôi. Thì chỉ cần cài JRE là đủ.

JDK

JDK: Java Development Kit

JDK = Development tools + JRE (JVM + Libraries)

Bên cạnh đó nó còn có công cụ phát triển (Development tools) như là:

Trình biên dịch (compiler)

Công cụ quản lý gói

Công cụ Debugger

Mọi người cần phát triển ứng dụng Java. thì cần phải cài đặt JDK
