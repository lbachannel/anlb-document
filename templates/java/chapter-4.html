Static

Static chỉ chơi với static

Dùng static cho những gì không của riêng object mà là của cả họ

Dùng static cho những gì là tiện ích/Utility/Helper chỉ cần một vùng ram

với mục đích cung cấp dịch vụ/service cho các object khác

Ví dụ Math.sqrt(), Math.abs(), String.format() đều là method static 

public class Student {
    private String id;
    public static String name; 

    public void showProfile() {...};
    public static void sayHi() {...};
}

public static String name;

Khi new mới 1 object. instance variable name sẽ được clone vào 1 vùng nhớ mới có tên là Student static zone

Khi thuộc tính name khởi tạo dùng static. Thì khi các obj được tạo mới. name của obj sau cùng sẽ được lưu vào static zone.

Hậu quả là thông tin name ăn theo người cuối cùng được tạo ra, do chỉ chung 1 vùng ram cho cái đám name

Đáng lẽ là của riêng từng ai đó, từng object sinh viên

public static void sayHi() {...};

Đối với method sayHi() đang dùng static. nếu obj a chấm thì sẽ không hiển thị sayHi().

Lý do là vì method sayHi() là của cả họ Student. Chứ không thuộc cá nhân mỗi đối tượng

Vì là của chung, nên sẽ chấm trực tiếp qua tên Class. Student.sayHi();

an.sayHi();

Khi chấm sẽ không hiển thị method sayHi(), phải tự gõ, không lỗi, nhưng không nên theo kiểu này

Thế giờ đã hiểu hai loại dấu chấm

Chấm trực tiếp qua tên Class tức là chạm sờ STATIC

Chấm qua con đường new tức là bạn đang hàm ý chơi với non-static

Vì nó là đồ riêng của từng object
